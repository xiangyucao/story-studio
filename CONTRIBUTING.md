# 参与贡献

感谢你帮助 Story Studio 变得更好。

1. Fork 仓库并从 `main` 创建功能分支。
2. 运行 `npm install`。
3. 修改后运行 `npm run check`。
4. Pull Request 请说明用户体验变化、数据迁移影响和验证方式。

请勿提交 API Key、`.env.local` 或 `data/story-studio.db`。数据库结构调整应保持旧数据可升级，并在 PR 中说明迁移策略。

## 设计原则

- 正式稿和结构化设定以本地数据库为唯一真相来源。
- AI 默认只生成提案，不应无确认覆盖正式稿。
- 所有模型提供方通过统一适配层接入。
- 核心管理与逻辑查询在没有模型时也必须可用。
